## OOP 设计原则

### SOLID 原则

1. **S - Single Responsibility** (单一职责)
   - 一个类应该只有一个改变的理由

2. **O - Open/Closed** (开放封闭)
   - 对扩展开放,对修改封闭

3. **L - Liskov Substitution** (里氏替换)
   - 子类应该能够替换父类

4. **I - Interface Segregation** (接口隔离)
   - 客户端不应依赖它不需要的接口

5. **D - Dependency Inversion** (依赖倒置)
   - 依赖抽象而不是具体实现

## 学习建议

1. 先理解概念,再编写代码
2. 画出类图帮助理解关系
3. 思考为什么要这样设计
4. 对比不同设计方案的优劣
5. 重构练习以改进设计
